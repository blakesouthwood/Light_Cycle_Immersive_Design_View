This program is currently in the concept phase. The more that
I think about it the more I envision it as an interactive design
tool to help think about the program from the human's and machine's
perspective which are both completely different. I believe this will
inevitably have several modes, levels, and perspectives as a tool.

Fred Brooks mentioned in the beginning of the Mythical Man Month
book "The essence of a software entity is a construct of interlocking concepts:"
This perspective that he described is incomprehensible by the computer
but it serves as an aid to humans to "undestand a program."
At the same time Edsger Dijkstra said, "We create machine behavior."
"I believe the hard part of building software to be the specification,
design, and testing of this conceptual construct, not the labor of
representing it and testing the fidelity of the representation." (Brooks 1987)

The importance of design is paramount and as programs evolve,
morph, and grow in complexity they become increasingly more
difficult to comprehend and grasp conceptually. The purpose
of this graphic game-like vision is to go inside a program
within the design (not the actual code) from a different representation
for better understanding by humans - the creators.


With each new program I built the complexity becomes crushing and there
are so many things to contend with and this way the intangible will
become tangible. This way it will be a useful guide to get unstuck
and fully comprehend what code is doing since it will be described
at the documentation level behaviorally (running mode) and
in the static boolean logic.

This would be an interactive graphic game like perspective of
going through a program at the function level and inside functions.
I was inspired by the movie "The Fantastic Voyage" when they went
through the blood stream of a patient.

This is to assist a programmer in designing and testing their
programs and components by a simulation of riding through the
program on a horizontal grid, with a map view in the corner
as if they are the Tron Cycle riding through the program at either
the function level from function to function on that path route
or at the lower level going through an algorithms flow of control.


There will be road signs and it's all for the human perspective
to aid the thinking process and for testing and to provide a way
of going into a program through a simulation from the design
view though it's just another representation and the computer
knows nothing about it. It's a radical interface view concept.


All my friends player games like Halo, Call of Duty, and other
first player movies and this way a programmer can be immersed
on the grid and in addition to my other concept of map view
this gets down to the boolean logic level to understand a function
better by going through the flow of control so it should offer
a unique perspective.


At the same time there will be a way of tracing the code flow of control
so a human programmer can review it kind of like Myst by clicking or movie
mode going through it like a driving game set at the speed they wish.

This was inspired by Wolfenstein and Doom and Quake of course and Myst.

Further it would show the design perspective and help to get the logic
right also hopefully.
